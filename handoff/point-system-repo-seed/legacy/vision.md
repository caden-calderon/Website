# Chromatic Vision

## Product Summary

Chromatic is a portfolio and creative system built around a point-based visual language.

The product still has two layers:

- an experiential layer: a stylized train compartment and adjacent cinematic scenes
- a functional layer: a real website that exposes the same portfolio content clearly

The key shift is that both layers should feel like they are made from points, surfels, dust, light, and sampled matter rather than ASCII characters or conventional shaded meshes.

## Experience Structure

### Experiential Layer

The original train-compartment idea still matters.

Core ingredients:

- a train compartment or adjacent carriage-like space
- a table between the visitor and the world/character focus
- props that can carry portfolio meaning
- a laptop or portal into the functional site
- motion outside the window or beyond the scene
- strong atmosphere: darkness, glow, travel, weather, vibration, quiet drama

### Functional Layer

There is still a normal website inside the product.

That website should be:

- clear
- recruiter-friendly
- accessible
- easier to scan than the experiential layer

But it should still inherit the point-system visual language through:

- hero treatments
- transitions
- previews
- selected project surfaces

### Character Layer

The original AI-guided character concept remains valid as a product/worldbuilding idea.

It is not a phase-1 implementation requirement.
The point-system runtime should leave room for:

- a seated character across the table
- animated performance derived from real motion
- later conversational or guided interactions

## Content Mirroring

The mirroring idea should survive the pivot.

Examples:

- object or scene detail in the world layer
- matching page, project, or section in the site layer

Illustrative mappings:

- business card or ticket -> contact / links
- paper, magazine, newspaper, or dossier -> resume / experience
- cards, chessboard, tabletop prop -> interactive project showcase
- window or passing scenery -> gallery / project browsing
- laptop or portal surface -> functional site

## Artistic Direction

The target look is closest to:

- persistent 3D point clouds
- colored stipple on black
- pointillist or surfel-like image construction
- visible structure without noisy temporal swimming
- a soft, luminous, museum-like presentation rather than glitchy particles

Important properties of the look:

- black or very dark grounds
- colored points with restrained bloom/glow
- stable identities over time
- subtle variation in size, opacity, and density
- legible silhouettes before dense realism
- motion that feels coherent, not re-randomized

## Core Product Decision

The rendering system is the product.

The website, train scene, project showcases, motion experiments, AR/VR ideas, and live inputs are all consumers of the same underlying point-system runtime.

This means the foundation should not be:

- a route-level website first
- a one-off post-process shader
- a pile of source-specific hacks

It should be:

- a canonical point representation
- a set of ingest pipelines
- a renderer/runtime that can present those points coherently across multiple applications

## What Chromatic Still Is

The original train-compartment concept remains valuable.

The world still includes:

- a compartment interior
- a table and interactive objects
- a seated character across from the viewer
- a laptop or portal into the functional site
- environmental storytelling through objects and scenes

The content mirroring idea also remains intact:

- train object or scene gesture
- matching functional website destination
- same portfolio information, different presentation mode

## What Changed

The old visual center of gravity was:

- ASCII
- braille
- glyph atlases
- text-art post-processing

The new center of gravity is:

- points
- surfels
- persistent samples on surfaces
- image/video particles
- scanned or reconstructed spatial data

The site itself should inherit this language:

- point-based reveals
- point-built thumbnails or scene cards
- point-cloud-like transitions
- point-native project showcases

But not everything should become points.
Body text, navigation, forms, and accessibility-critical UI should remain DOM-native and readable.

## Supported Source Types

The long-term point system should eventually support:

- Blender/glTF models
- animated or skinned characters
- still images
- pre-recorded video
- webcam/live video
- iPhone/iPad LiDAR captures
- other point-cloud or spatial scan formats
- VR and AR presentation layers

This does not mean all of them should be built at once.
It means the core representation and architecture should not block them.

## Temporal Coherence Principle

Stable motion is a first-class artistic requirement.

If dots are regenerated independently every frame, the result becomes shimmer and noise.
The preferred approaches are:

- persistent points attached to 3D surfaces
- particles with stable IDs sampling changing media
- cached or reconstructable samples with explicit correspondence over time

For Chromatic, 3D-attached points are the primary path because they best support:

- character animation
- camera motion
- consistent silhouettes
- Blender-authored scenes

## Non-Goals For Phase 1

Do not try to solve everything immediately.

Phase 1 is not:

- a full portfolio site
- a full train experience
- native iOS LiDAR capture
- a WebXR product
- a universal import stack
- a final visual design system

Phase 1 is a proof that the point-system runtime can carry the look and motion quality the project needs.
